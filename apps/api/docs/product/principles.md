# Product Principles

> **Status:** Approved · **Owner:** Product / feature lead · **Reviewers:** Backend, Frontend, Mobile, Design · **Last updated:** 2026-07-21
> **Derived from: Product Model §5 (Core principles) and the Framing section (what kind of product this is).**

The principles that constrain every product decision. These are the ones defined in the
[Product Model](PRODUCT_MODEL.md); this document only clarifies how each is applied. It
adds no new principle — a new principle requires amending the model first (§Governance).

## Framing principles

From the Product Model's framing:

1. **An AI-powered learning platform, not a collection of features** — evaluate every
   proposal by whether it strengthens the one connected system, not a standalone tool.
2. **AI is a mentor and coach, not an answer generator** — the platform teaches and
   questions; it never hands over answers for real-interview use.
3. **One connected ecosystem** — features share data and value; prefer designs where a
   module's output feeds another (see [`feature-map.md`](feature-map.md)).
4. **The Product Model is the single source of truth** — product docs derive from it.

## Core principles (§5)

- **Learn by doing** — practice is the product; a feature that turns Level Up into
  passive consumption works against this.
- **Practice before theory** — let the user rehearse, then fill gaps; don't gate
  practice behind lectures.
- **AI is a mentor, not an answer machine** — coaches and questions; never supplies
  real-interview answers, and hides scores mid-conversation.
- **Personalized learning** — adapt to the individual; per-answer performance is
  captured from day one to enable this.
- **Continuous feedback** — every answer must produce a signal the user can act on.

How these principles shape the *why* behind each experience is expanded in
[`philosophy.md`](philosophy.md).
