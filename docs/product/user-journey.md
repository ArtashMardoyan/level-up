# User Journey

> **Derived from: Product Model §6 (The complete user journey).**

The stages a user moves through. The canonical journey diagram lives in the
[Product Model §6](PRODUCT_MODEL.md#6-the-complete-user-journey); this document expands
each stage. It adds no stage beyond §6.

## Stages

1. **Discover** — the user finds Level Up.
2. **Register** — they create an account.
3. **Assessment** *(future)* — an upfront gauge of where they stand (defined as future in §6).
4. **Study a track** — they work through a curated question bank for their track.
5. **AI mock interview** — they rehearse: the AI asks one question at a time.
6. **Scored feedback + model answers** — each answer is scored on the rubric and coached.
7. **Review weak areas** — they see where they fell short.
8. **Improve** — they retry, or use the Dictionary to sharpen communication.
9. **Track progress + earn badges** — progress is visible and rewarded.
10. **Interview-ready → Career growth** — the outcome the platform exists to produce.

## The loop that matters

The **Improve → Study/Practice** loop (§6) is where the platform's compounding value
lives: every pass through it is meant to raise the next score. Every stage but
*Assessment* is live today. Which module powers each stage is described in
[`feature-map.md`](feature-map.md).
